# 专业解读卡片模板（major_card_template）

> 阶段二，AI 读取 T1 后，对用户在 T1 里「**是否考虑？=是**」的每个专业类各生成一张卡，组成"专业解读"智能文档（用 `create_smartcanvas_by_mdx` 或 `create_smartcanvas_by_markdown`）。生成时可结合该行用户填的「该专业类吸引我的原因？」做个性化匹配点。

## 卡片 Markdown 模板

> **排版口径**：字段内**只写事实 + 引用编号**（如"（来源见末尾汇总 #1）"），完整 URL 三件套统一在卡片末尾「数据来源汇总」段中按编号列出，避免视觉冗余。

### {专业类名称}

- **就业与薪资**：{方向概览；薪资梯队}（来源见末尾汇总 #1）
- **是否需读研**：{本科可就业 / 建议读研 / 多数核心岗要求研究生}（来源见末尾汇总 #2）
- **院校层次敏感度**：{高—很看平台牌子 / 中 / 低—更看个人能力}（主观判断，仅供参考；如有支撑材料见末尾汇总 #3）
- **培养方案**：{主要课程与能力要求，如对数学、英语的要求}（来源见末尾汇总 #4）
- **选科匹配**：{结合用户选科：可报 / 部分受限（说明原因）}（用户选科信息来自阶段一 A/B 卡，无需 URL；选科要求规则见末尾汇总 #5）
- **数据来源汇总**：
  - #1 [{页面名 1}]({URL 1}) · 采集年份：{YYYY} · 采集时间：{YYYY-MM-DD}
  - #2 [{页面名 2}]({URL 2}) · 采集年份：{YYYY} · 采集时间：{YYYY-MM-DD}
  - #3 [{页面名 3 / 可省略}]({URL 3}) · 采集年份：{YYYY} · 采集时间：{YYYY-MM-DD}
  - #4 [{页面名 4}]({URL 4}) · 采集年份：{YYYY} · 采集时间：{YYYY-MM-DD}
  - #5 [{页面名 5}]({URL 5}) · 采集年份：{YYYY} · 采集时间：{YYYY-MM-DD}

> **宁缺毋滥**：找不到可信 URL 的事实陈述请直接省略或改为"待核实"，**严禁**编造 URL，**严禁**用机构首页 URL（如 `https://gaokao.chsi.com.cn/`）占位替代具体页面 URL。用户在阶段一/二填写的个人信息（选科、分数、偏好等）来源即用户本人，无需也不应挂 URL。
> **编号一致性**：字段内"#N"必须能在末尾汇总段一一对应；若某条事实没有可信 URL，字段处直接写"待核实"，**不要**写"（来源见末尾汇总 #X）"指向一个不存在的编号。

## 填充示例

### 电子信息类

- **就业与薪资**：芯片、通信、嵌入式、互联网研发等方向，整体薪资偏高，头部企业与硕士起薪更高（来源见末尾汇总 #1）
- **是否需读研**：本科可就业；芯片设计、算法等方向建议读研，常规开发岗本科即可（来源见末尾汇总 #2）
- **院校层次敏感度**：中—大厂校招看平台，但个人项目与能力同样关键（主观判断，仅供参考）
- **培养方案**：电路、信号、编程，数学要求较高；英语主要用于阅读文献（来源见末尾汇总 #3）
- **选科匹配**：物理类、含化学者覆盖更广；纯历史类通常不可报（用户选科信息来自阶段一 A/B 卡；选科要求规则见末尾汇总 #4）
- **数据来源汇总**：
  - #1 [电子科技大学 2024 届毕业生就业质量报告](https://www.uestc.edu.cn/xxxxxx) · 采集年份：2024 · 采集时间：2026-06-09
  - #2 [阳光高考电子信息类专业介绍](https://gaokao.chsi.com.cn/zyk/zybk/specialityDetail.action?specialityId=xxx) · 采集年份：2025 · 采集时间：2026-06-09
  - #3 [电子科技大学电子信息工程培养方案](https://www.uestc.edu.cn/xxxxxx) · 采集年份：2025 · 采集时间：2026-06-09
  - #4 [阳光高考选科要求查询](https://gaokao.chsi.com.cn/xxxxxx) · 采集年份：2026 · 采集时间：2026-06-09

## 字段要点

- 主观判断（薪资梯队、层次敏感度）必须标"仅供参考、建议交叉验证"，并附来源年份；如能找到具体研究/报告 URL 则放入末尾汇总，否则直接标"主观判断"，**不强行编号挂空 URL**。
- 选科匹配结合阶段一 A/B 信息卡里的用户选科得出；不匹配要明确标出。**用户选科本身不挂 URL**（来源即用户），选科要求规则在末尾汇总段挂阳光高考或本省考试院 URL。
- 查不到可靠数据的字段标"待核实"，**不编造，也不用首页 URL 占位**。
- 末尾汇总段所有 URL 必须是**具体页面级**链接（能直接跳到事实所在页），**严禁**只贴机构首页或栏目首页。

## 统一底注（每份专业解读文档末尾）

**渲染要求**：以下文案在交付的腾讯文档 / 报告里必须**整段加粗**呈现，**不要**包裹 `>` 引用块（截图实测引用块在腾讯文档里渲染成灰色弱化样式，反而让免责声明视觉权重下降）。直接作为一段加粗正文紧贴在标题下方即可。

```markdown
**本解读由 AI 基于公开信息整理，仅供参考；就业、薪资、培养方案等会随年份和院校变化，请以院校官方招生与培养方案、就业质量报告为准。本内容基于该同学填写的 T1 兴趣与个人情况生成。**
```
