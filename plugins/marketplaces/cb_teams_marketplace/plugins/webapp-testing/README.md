# Web 应用测试助手插件

智能引导式 Web 应用测试助手，帮助你快速启动、配置和测试 Web 应用。自动分析项目结构，检测运行环境，配合 `agent-browser` 进行浏览器自动化测试。

## 安装

```bash
# 通过 CodeBuddy 插件市场安装
/plugin install webapp-testing@cb-teams
```

## 功能说明

安装此插件后，当你提到测试 Web 应用相关的需求时，CodeBuddy 会自动启用测试助手流程，逐步引导你完成以下步骤：

1. **检查项目代码** — 确认当前目录是否有项目，没有则提示先 clone
2. **分析项目结构** — 自动识别技术栈、推断启动命令，让你确认配置
3. **启动应用** — 可选由 Agent 帮你执行启动命令
4. **判断测试方式** — 纯 Web 应用使用浏览器测试，API 服务使用命令行测试
5. **环境与登录检测** — 检测操作系统，处理登录场景（有头浏览器）
6. **执行测试** — 使用 `agent-browser` 进行自动化浏览器测试

## 使用示例

### 场景 1：测试 React 前端项目

```
User: 帮我测试一下这个项目

CodeBuddy:
1. 检测到 package.json，识别为 React + Vite 项目
2. 推断启动命令为 npm run dev，端口 5173
3. 询问用户确认，是否需要额外配置
4. 帮助启动开发服务器
5. 使用 agent-browser 打开页面，截图分析
6. 报告页面结构和潜在问题
```

### 场景 2：测试需要登录的后台系统

```
User: 帮我测试后台管理系统，需要登录

CodeBuddy:
1. 分析项目结构，确认启动方式
2. 检测当前为 macOS 环境，支持有头浏览器
3. 启动 agent-browser（--headed 模式）
4. 打开登录页面，等待用户手动完成登录
5. 登录完成后继续自动化测试
```

### 场景 3：Linux 环境下测试

```
User: 帮我测试这个 Web 应用，需要登录

CodeBuddy:
1. 检测到当前为 Linux 环境，无图形界面
2. 告知用户无法使用有头浏览器
3. 建议用户提供 cookie/token 注入方式
4. 或建议使用 API 登录获取 session
5. 配置完成后使用无头浏览器进行测试
```

## 前置依赖

- **agent-browser skill**：浏览器自动化测试需要安装此 skill。插件会自动检测，未安装时会提示引导安装。
- 对于纯 API 测试场景，不需要 agent-browser。

## 支持的项目类型

| 项目类型 | 测试方式 | 说明 |
|---------|---------|------|
| React / Vue / Next.js 等前端项目 | agent-browser 浏览器测试 | 自动识别并使用浏览器测试 |
| 全栈项目（前端 + 后端） | agent-browser 浏览器测试 | 测试前端页面交互 |
| 纯后端 API 服务 | curl / 命令行测试 | 不需要 agent-browser |
| 静态 HTML 页面 | agent-browser 浏览器测试 | 直接打开页面测试 |

## 环境兼容性

| 操作系统 | 无头浏览器 | 有头浏览器（登录场景） |
|---------|-----------|---------------------|
| macOS | 支持 | 支持 |
| Windows | 支持 | 支持 |
| Linux（有桌面） | 支持 | 支持 |
| Linux（无桌面） | 支持 | 不支持，需替代方案 |
