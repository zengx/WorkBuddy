# 通用约定与分析基础

所有分析任务共享的规范和方法论。

## 数值精度

- 颜色：`#RRGGBB`
- 尺寸：`"28px"`
- 不确定时 `~` 前缀（`"~14px"`），但不要过度标注

## 坐标系

百分比归一化坐标系：原点 `(0,0)` 左上，`(100,100)` 右下，精度 1 位小数。

```json
{"bbox": {"x": 0, "y": 0, "width": 16.7, "height": 100}}
```

每个 section 和关键 element 必须输出 `bbox`。

## 尺寸格式

`width`/`height`：`"240px"` / `"fill"` / `"auto"` / `"33.3%"`

## 空值

无法识别 → `null`；不存在 → 省略字段。

## 语言

key 英文，描述性 value 根据用户语言（中文用户 → 中文描述）。

## bbox 估算（4 步法）

**Step 1 — 顶层划分**：互不重叠地划分页面，width 之和 = 100。

```
sidebar_main: Sidebar w:16.7 + Main w:83.3 = 100 ✓
```

**Step 2 — 区域内定位**：在父 bbox 内，按布局方向依次排列。已知 px 尺寸换算百分比：`64px / 900px = 7.1%`。

**Step 3 — 设计系统约束**：等分网格均匀分配、padding 偏移一致、flex space-between 首尾紧贴。

**Step 4 — 交叉验证**：覆盖完整（无遗漏）、无异常重叠、子在父内、等分元素宽高相同。

### bbox 速查

| 场景 | bbox |
|------|------|
| 全宽导航栏 64px/900px | `{x:0, y:0, w:100, h:7.1}` |
| 左侧栏 240px/1440px | `{x:0, y:0, w:16.7, h:100}` |
| 主内容区（240px 侧栏后） | `{x:16.7, y:0, w:83.3, h:100}` |
| iOS Tab Bar 49pt/812pt | `{x:0, y:94, w:100, h:6}` |
| 3 等分卡片 gap24px/1200px 容器 | 每卡片 w:26.7 |

## 常见陷阱

**颜色**：渐变色 → 给起止色；半透明 → 给最终呈现色；暗色模式 primary 更亮。

**布局**：滚动截断 → 注明可见部分；弹窗 → 分别描述 Modal 和底层；只描述当前 Tab。

**字号**：注意 Retina 2x/3x 的物理/逻辑像素区别；中文同字号视觉更大。
