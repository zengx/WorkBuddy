# 个股深度分析

---

## 适用边界

**使用本技能的情形：**
- 明确的单一公司名称或股票代码
- 涉及个股的投资逻辑、基本面、财务、技术分析、资金面、股票速览、公司战略、财务报表分析、同业比较、同行比较、研究机构观点、重大事件、近期动态、股价异动、投资者问答

**不使用本技能的情形：**
- 聚焦大盘指数整体表现 
- 聚焦宏观政策、利率、流动性背景
- 要求从市场主线发现方向、寻找候选标的池 
- 需要盘中实时异动提醒
- 只需要一个数据点(现价/PE/市值等)
- 筛选一批股票

---

## 数据获取

本技能通过 finance-data plugin 获取所需的金融数据(行情、财务、资金流向等)。

**获取原则：**
1. 根据分析主题，获取该公司相关的行情数据、财务数据、资金流向、机构持仓等结构化数据
2. 如某项数据暂时无法获取，可调整查询维度或换用相近指标
3. 问句涉及多个维度时，分别获取各维度所需数据

---

## 第一步：确认公司名称或股票代码

从问句、输入内容中提取公司名称或股票代码。

- 提取到明确名称或代码，直接进入数据获取
- 如果确认不了明确的股票名称或代码，可以返回合适的反问问句要求输入源进行信息补充。

---

## 第二步：确定主题领域

从问句、输入内容中通过你自己的理解，确定主题领域。该主题领域必须属于后续词汇(以顿号分割)的一项或多项：投资逻辑、基本面、财务、技术分析、资金面、股票速览、公司战略、财务报表分析、同业比较、同行比较、研究机构观点、重大事件、近期动态、股价异动、投资者问答。确定方式包括直接命中，或高相似度关联。

---

## 第三步：获取数据

根据确定的公司和主题领域，通过 finance-data plugin 获取所需数据。

不同主题领域对应的数据获取方向：

| 主题领域 | 需要获取的数据类型 |
|---------|--------------|
| 投资逻辑 | 公司基本信息、财务指标、行业地位 |
| 基本面分析 | 营收利润趋势、行业对比、竞争格局 |
| 财务分析 | 利润表、资产负债表、现金流量表、财务指标 |
| 技术分析 | 历史行情、成交量、均线等技术指标 |
| 资金面解读 | 资金流向、北向资金、主力动向 |
| 公司速览 | 公司基本信息、核心财务、估值水平 |
| 同业比较 | 同行业公司财务与估值横向对比 |
| 近期动态/异动 | 近期行情、公告、新闻事件 |

问句涉及多个维度时，分别获取各维度所需数据并行处理。

如某项数据暂时无法获取，可调整查询维度或换用相近指标。

---

## 第四步：输出

根据获取的数据，结合专业分析框架，针对问句、输入内容进行回答。
输出的内容应该以合适的统一的markdown形式给出。
输出字数应该与获取的数据量和分析深度相匹配。除非问句、输入内容特别要求，否则总字数不应该超过1500字。

## 第五步：输出规范

**强制要求：**
- 分析内容应基于获取的数据和专业分析框架
- 引用机构观点统一标注"据机构观点"，管理层表态标注"据管理层公开表态"
- 取不到的指标直接省略，不在正文中说明原因
- 正文结束即为输出结尾，不附加任何说明性文字

**严禁出现：**
- "受数据限制……"、"工具未返回……"
- "以上仅供参考"、"免责声明"、"几点说明"、"注："
- 任何解释工具调用状态的文字
