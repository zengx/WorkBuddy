选择您的首选平台开始使用:

**步骤 1: 安装 CodeBuddy Code**

**macOS:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows:**
```powershell
irm https://claude.ai/install.ps1 | iex
```

在 CodeBuddy Code 中,运行以下命令:
```bash
/plugin marketplace add K-Dense-AI/claude-scientific-skills
```

**步骤 3: 安装 Plugin**

**选项 A: 直接安装(最快)**
```bash
/plugin install scientific-skills@claude-scientific-skills
```

**选项 B: 交互式安装**
1. 在 CodeBuddy Code 中运行 `/plugin`
2. 选择 **Browse and install plugins**
3. 选择 **claude-scientific-skills** marketplace
4. 选择 **scientific-skills**
5. 点击 **Install now**

**就这样!** 当您描述科学任务时,CodeBuddy 将自动使用适当的技能。

**管理您的 Plugin:**
```bash
# 检查已安装的 plugins

# 更新 plugin 到最新版本
```

## ⚙️ 前提条件

- **Python**: 3.9+ (推荐 3.12+ 以获得最佳兼容性)
- **uv**: Python 包管理器(安装技能依赖项所需)
- **客户端**: CodeBuddy Code、Cursor 或任何兼容 MCP 的客户端
- **系统**: macOS、Linux 或带有 WSL2 的 Windows
- **依赖项**: 由各个技能自动处理(查看 `SKILL.md` 文件了解具体要求)

### 安装 uv

技能使用 `uv` 作为包管理器来安装 Python 依赖项。使用适用于您操作系统的说明安装它:

**macOS 和 Linux:**
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**Windows:**
```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**替代方法(通过 pip):**
```bash
pip install uv
```

安装后,通过运行以下命令验证它是否工作:
```bash
uv --version
```

有关更多安装选项和详细信息,请访问 [官方 uv 文档](https://docs.astral.sh/uv/)。

## 🔧 故障排除

### 常见问题

**问题: CodeBuddy Code 中的技能未加载**
- 解决方案: 确保您已安装最新版本的 CodeBuddy Code
- 验证 plugin 是否已安装: `/plugin → Manage Plugins`
- 尝试重新安装: `/plugin uninstall scientific-skills@claude-scientific-skills` 然后 `/plugin install scientific-skills@claude-scientific-skills`
- 如需要,重新添加 marketplace: `/plugin marketplace add K-Dense-AI/claude-scientific-skills`

**问题: 缺少 Python 依赖项**
- 解决方案: 检查特定的 `SKILL.md` 文件了解所需的包
- 安装依赖项: `uv pip install package-name`

**问题: API 速率限制**
- 解决方案: 许多数据库都有速率限制。查看特定数据库文档
- 考虑实施缓存或批量请求

**问题: 身份验证错误**
- 解决方案: 某些服务需要 API 密钥。查看 `SKILL.md` 了解身份验证设置
- 验证您的凭据和权限

**问题: 示例过时**
- 解决方案: 通过 GitHub Issues 报告问题
- 查看官方包文档了解更新的语法

## ❓ FAQ

### 一般问题

**问: 这是免费使用的吗?**  
答: 是的!本仓库采用 MIT 许可证。但是,每个单独的技能都有自己的许可证,在其 `SKILL.md` 文件的 `license` 元数据字段中指定——请务必查看并遵守这些条款。

**问: 为什么所有技能都分组到一个 plugin 中,而不是单独的 plugins?**  
答: 我们相信 AI 时代的优秀科学本质上是跨学科的。将所有技能捆绑到一个 plugin 中,使您(和 CodeBuddy)可以轻松跨越领域——例如,在一个工作流程中结合基因组学、化学信息学、临床数据和机器学习——而无需担心要安装或连接哪些单独的技能。

**问: 我可以将此用于商业项目吗?**  
答: 仓库本身采用 MIT 许可证,允许商业使用。但是,单独的技能可能有不同的许可证——查看每个技能 `SKILL.md` 文件中的 `license` 字段以确保符合您的预期用途。

**问: 所有技能都有相同的许可证吗?**  
答: 不。每个技能都有自己的许可证,在其 `SKILL.md` 文件的 `license` 元数据字段中指定。这些许可证可能与仓库的 MIT 许可证不同。用户有责任查看并遵守他们使用的每个单独技能的许可证条款。

**问: 这多久更新一次?**  
答: 我们定期更新技能以反映包和 API 的最新版本。主要更新在发布说明中宣布。

**问: 我可以将其与其他 AI 模型一起使用吗?**  
答: 技能针对 CodeBuddy 进行了优化,但可以适应支持 MCP 的其他模型。MCP 服务器适用于任何兼容 MCP 的客户端。

### 安装与设置

**问: 我需要安装所有 Python 包吗?**  
答: 不!只安装您需要的包。每个技能在其 `SKILL.md` 文件中指定了要求。

**问: 如果技能不起作用怎么办?**  
答: 首先查看 [故障排除](#troubleshooting) 部分。如果问题仍然存在,请在 GitHub 上提交问题,并提供详细的重现步骤。

**问: 技能可以离线工作吗?**  
答: 数据库技能需要互联网访问来查询 API。安装 Python 依赖项后,包技能可以离线工作。

### 贡献

**问: 我可以贡献自己的技能吗?**  
答: 当然可以!我们欢迎贡献。查看 [贡献](#contributing) 部分了解指南和最佳实践。

**问: 如何报告错误或建议功能?**  
答: 在 GitHub 上打开问题,并提供清晰的描述。对于错误,请包括重现步骤以及预期行为与实际行为的对比。
