# /weixin-minigame-helper:preview — 启动小游戏预览

## 执行步骤

### 1. 确定游戏目录

- 如果用户提供了参数（游戏目录路径），使用该路径
- 否则，在当前工作目录中搜索 `game.js` 文件：
  - 先检查当前目录根是否有 `game.js`
  - 再递归搜索子目录
- 如果找不到 `game.js`，告诉用户："未找到 game.js 文件。请确保当前目录是微信小游戏项目，或指定游戏目录路径。"

### 2. 启动预览

- 加载 `weixin-minigame-helper` skill
- 调用 MCP 工具 `run_game`，传入游戏目录的**绝对路径**
- 等待返回预览 URL

### 3. 打开浏览器

- 使用 `preview_url` 工具打开返回的预览 URL
- 告诉用户：
  - 预览已启动
  - 预览地址
  - 修改代码后会自动检测变更，刷新浏览器即可看到最新效果

### 4. 错误处理

- 如果 `run_game` 失败，检查错误信息并给出修复建议
- 常见问题：
  - `game.js` 不存在 → 提示用户创建入口文件
  - 端口被占用 → 工具会自动选择可用端口，通常不需要用户干预
