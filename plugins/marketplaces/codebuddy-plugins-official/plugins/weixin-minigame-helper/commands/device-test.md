# /weixin-minigame-helper:device-test — 真机测试

## 执行步骤

### 1. 确定游戏目录

- 如果用户提供了参数，使用该路径
- 否则搜索当前工作目录中的 `game.js`
- 路径必须转为绝对路径

### 2. 确保预览服务已启动

- 先调用 `run_game` 启动预览服务（如果尚未启动）
- 预览服务需要先运行，这样用户才能在预览页面看到二维码弹窗

### 3. 执行真机预览

- 调用 MCP 工具 `real_device_preview`，传入游戏目录绝对路径
- 等待结果

### 4. 处理结果

**成功的情况：**
- 告诉用户："真机预览二维码已生成！请在浏览器预览页面查看弹出的二维码，用微信扫码即可在真机上测试。"

**配置缺失的情况（configMissing: true）：**
- 告诉用户需要配置微信 AppID 和代码上传密钥
- 引导配置方式：

  **方式一：通过预览页面配置（推荐）**
  1. 在浏览器预览页面点击右上角 ⚙️ 配置按钮
  2. 在弹出面板中填写：
     - 微信 AppID（在微信公众平台 mp.weixin.qq.com 获取）
     - 代码上传密钥（微信公众平台 → 管理 → 开发管理 → 开发设置 → 小程序代码上传 → 生成密钥）
  3. 保存后重新执行此命令

  **方式二：通过环境变量配置（适合 CI/CD）**
  ```bash
  export WECHAT_APPID=wxXXXXXXXX
  export WECHAT_PRIVATE_KEY_PATH=/path/to/private.key
  ```

**其他错误：**
- 显示错误信息并给出可能的修复建议
- IP 白名单问题：提示用户在微信公众平台添加当前机器的 IP
