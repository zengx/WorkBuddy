## 目标

根据参考节点（模块节点、场景节点或需求节点）及关联的需求信息分析并生成相应的测试点，测试点节点必须是参考节点的子节点。
请综合分析参考节点的功能特点，识别出需要验证的具体测试点。

**参考节点列表**

**需求分析内容**
<issue_analysis>

```
${issue_analysis}
```

</issue_analysis>

## 生成规则

1. 参考节点必须有有效的uid，否则直接报错
2. 只生成测试点（TEST_POINT）
3. 生成的测试点必须与参考节点关联
4. 每个测试点必须有明确的验证目标和可执行性
5. 测试点应具备原子性，不可再分解

## 输出要求

**🚨 关键规则 - 确保输出稳定性**：

1. **强制使用标准JSON格式输出**
   - 必须输出严格的JSON数组格式
   - 所有字段必须使用双引号
   - 特殊字符必须正确转义（引号用 `\"`，换行符用 `\n`）
   - 不允许使用YAML格式

2. **输出格式验证**
   - 输出必须是可直接解析的JSON数组
   - 每个测试点节点必须包含所有必需字段：`uid`, `name`, `description`, `kind`, `parent_uid`
   - 必须为每个节点生成唯一的 `uid` 字段

3. **禁止行为**：
   - ❌ 不要创建任何 Python 脚本
   - ❌ 不要使用YAML格式输出
   - ❌ 不要在JSON中添加注释
   - ❌ 不要输出除JSON数组外的任何其它内容（除了简短确认信息）

4. 直接输出到临时文件中

## 测试点类型

根据测试理论和验证需要，可生成以下类型的测试点：

### 1. 功能验证测试点

- **正向功能测试点** - 验证功能正常执行路径
- **反向功能测试点** - 验证功能异常处理能力
- **业务规则测试点** - 验证业务逻辑和规则正确性

### 2. 数据验证测试点

- **输入验证测试点** - 验证输入数据的有效性检查
- **输出验证测试点** - 验证输出数据的正确性和完整性
- **数据转换测试点** - 验证数据处理和转换逻辑

### 3. 边界条件测试点

- **边界值测试点** - 验证边界值和临界值处理
- **等价类测试点** - 验证等价类划分的代表性数据
- **极限值测试点** - 验证系统在极限条件下的表现

### 4. 状态验证测试点

- **状态转换测试点** - 验证状态机的转换逻辑
- **状态保持测试点** - 验证状态的持久性和一致性
- **并发状态测试点** - 验证并发访问下的状态管理

### 5. 接口交互测试点

- **接口调用测试点** - 验证接口的调用和响应
- **参数传递测试点** - 验证参数的传递和解析
- **协议兼容测试点** - 验证协议的兼容性和标准性

### 6. 性能验证测试点

- **响应时间测试点** - 验证系统响应时间要求
- **吞吐量测试点** - 验证系统处理能力
- **资源消耗测试点** - 验证系统资源使用情况

### 7. 安全验证测试点

- **权限控制测试点** - 验证访问权限和授权机制
- **数据安全测试点** - 验证数据加密和保护措施
- **攻击防护测试点** - 验证系统安全防护能力

## 测试点设计原则

1. **原子性** - 每个测试点验证单一明确的功能点
2. **可执行性** - 测试点必须可以被具体执行和验证
3. **独立性** - 测试点之间相互独立，不存在依赖关系
4. **可重复性** - 测试点可以重复执行并得到一致结果
5. **可验证性** - 测试点有明确的通过/失败判断标准

## 测试点生成策略

### 基于模块节点生成测试点

- 分析模块的核心功能和关键逻辑
- 识别模块的输入输出和处理流程
- 考虑模块的异常情况和边界条件
- 覆盖模块的各种使用场景和配置

### 基于场景节点生成测试点

- 分解场景中的具体验证步骤
- 识别场景中的关键验证点
- 考虑场景执行过程中的状态变化
- 覆盖场景的正常和异常分支

### 基于需求节点生成测试点

- 解析需求的具体功能要求
- 识别需求的验收标准和质量要求
- 分析需求的约束条件和限制
- 考虑需求实现的技术细节

## 标准输出格式

**必须严格遵循以下JSON格式**：

```json
[
  {
    "uid": "test_point-abc123",
    "name": "验证登录页面正常显示",
    "description": "验证登录页面能够正常加载，包含用户名、密码输入框和登录按钮",
    "kind": "TEST_POINT",
    "parent_uid": "scene_12345"
  },
  {
    "uid": "test_point-def456",
    "name": "验证用户名输入有效性",
    "description": "验证用户名输入框能够正确接受有效的用户名格式",
    "kind": "TEST_POINT",
    "parent_uid": "scene_12345"
  },
  {
    "uid": "test_point-ghi789",
    "name": "验证密码输入安全性",
    "description": "验证密码输入框能够正确隐藏密码字符，显示为*号",
    "kind": "TEST_POINT",
    "parent_uid": "scene_12345"
  }
]
```

## 字段说明

| 字段          | 类型   | 必填 | 说明                                        |
| ------------- | ------ | ---- | ------------------------------------------- |
| `uid`         | string | ✅   | 节点唯一标识，格式：`test_point-{随机字符}` |
| `name`        | string | ✅   | 测试点名称，简洁明确                        |
| `description` | string | ✅   | 测试点详细描述                              |
| `kind`        | string | ✅   | 固定值：`TEST_POINT`                        |
| `parent_uid`  | string | ✅   | 必须使用 `ref_nodes` 参数中提供的节点 uid   |

## 特殊字符处理规则

在JSON字符串中，以下字符必须转义：

- 双引号：`"` → `\"`
- 反斜杠：`\` → `\\`
- 换行符：不允许使用真实换行，用 `\n` 或用逗号/分号分隔
- 制表符：`\t`

**示例**：

```json
{
  "name": "验证\"密码\"输入安全性",
  "description": "验证密码输入框能够正确隐藏密码字符，显示为*号或其它安全符号"
}
```
