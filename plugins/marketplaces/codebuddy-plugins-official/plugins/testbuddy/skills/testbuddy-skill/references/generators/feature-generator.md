  Args:
      inputs: {

## 目标

根据需求节点及需求详情归纳并识别出测试的功能模块。
**参考节点**
<ref_node>
${ref_nodes}
</ref_node>

**需求分析内容**
<issue_detail>

```
${issue_analysis}
```

</issue_detail>

## 约束

1. 参考节点必须有效，否则直接报错
2. 需求详情必须完整，否则直接报错
3. 只生成FEATURE类型的模块节点
4. 生成的模块必须与需求内容相关，忽略无关信息
5. 每个模块必须有明确的测试范围和职责

## 模块类型

根据需求特点，可生成以下类型的功能模块：

1. **核心业务模块** - 主要业务功能和流程
2. **基础服务模块** - 支撑性服务和组件
3. **接口交互模块** - API和数据交互
4. **用户界面模块** - UI交互和展示
5. **数据处理模块** - 数据存储和处理逻辑
6. **安全控制模块** - 权限和安全相关功能

## 输出要求

**🚨 关键规则 - 确保输出稳定性**：

1. **强制使用标准JSON格式输出**
   - 必须输出严格的JSON数组格式
   - 所有字段必须使用双引号
   - 特殊字符必须正确转义（引号用 `\"`，换行符用 `\n`）
   - 不允许使用YAML格式

2. **输出格式验证**
   - 输出必须是可直接解析的JSON数组
   - 每个模块节点必须包含所有必需字段：`uid`, `name`, `description`, `kind`, `parent_uid`
   - 必须为每个节点生成唯一的 `uid` 字段

3. **禁止行为**：
   - ❌ 不要创建任何 Python 脚本
   - ❌ 不要使用YAML格式输出
   - ❌ 不要在JSON中添加注释
   - ❌ 不要输出除JSON数组外的任何其它内容（除了简短确认信息）

4. 直接输出到临时文件中

## 标准输出格式

**必须严格遵循以下JSON格式**：

```json
[
  {
    "uid": "feature-abc123",
    "name": "用户身份验证模块",
    "description": "负责用户注册过程中的身份验证，包括用户名唯一性检查、邮箱验证、手机号验证等功能",
    "kind": "FEATURE",
    "parent_uid": "story_12345"
  },
  {
    "uid": "feature-def456",
    "name": "密码安全控制模块",
    "description": "负责用户密码相关的安全控制，包括密码强度校验、密码确认、密码加密存储等功能",
    "kind": "FEATURE",
    "parent_uid": "story_12345"
  }
]
```

## 字段说明

| 字段          | 类型   | 必填 | 说明                                      |
| ------------- | ------ | ---- | ----------------------------------------- |
| `uid`         | string | ✅   | 节点唯一标识，格式：`feature-{随机字符}`  |
| `name`        | string | ✅   | 模块名称，简洁明确                        |
| `description` | string | ✅   | 模块详细描述                              |
| `kind`        | string | ✅   | 固定值：`FEATURE`                         |
| `parent_uid`  | string | ✅   | 必须使用 `ref_nodes` 参数中提供的节点 uid |

## 特殊字符处理规则

在JSON字符串中，以下字符必须转义：

- 双引号：`"` → `\"`
- 反斜杠：`\` → `\\`
- 换行符：不允许使用真实换行，用 `\n` 或用逗号/分号分隔
- 制表符：`\t`

**示例**：

```json
{
  "name": "用户\"身份\"验证模块",
  "description": "负责用户注册过程中的身份验证，包括：用户名唯一性检查、邮箱验证、手机号验证等功能"
}
```
