# 用例腐化诊断规则

## 腐化检测维度

| 维度             | 对应字段                | 腐化判定标准                                           |
| ---------------- | ----------------------- | ------------------------------------------------------ |
| **名称腐化**     | `name`                  | 名称不清晰、不符合约束条件中的功能点描述、包含过时术语 |
| **描述缺失**     | `description`           | 描述为空或与约束条件不匹配                             |
| **前置条件过时** | `pre_conditions`        | 前置条件与当前系统状态不符、引用了已废弃的环境/配置    |
| **步骤陈旧**     | `steps[]` / `step_text` | 步骤中的操作路径变更、交互方式变化、预期结果与实际不符 |
| **优先级失准**   | `priority`              | 功能重要性变化导致优先级不合理                         |
| **重复冗余**     | 多用例综合              | 多个用例覆盖相同功能点                                 |

## 腐化严重度分级

| 级别     | 说明                                   | 处理方式       |
| -------- | -------------------------------------- | -------------- |
| **严重** | 用例已完全失效（功能已删除/重构）      | 标记删除       |
| **较高** | 步骤/预期结果与实际严重不符            | 重写步骤和预期 |
| **中等** | 部分字段需要更新（名称/描述/前置条件） | 更新相应字段   |
| **轻微** | 仅优先级或表述需微调                   | 微调           |
| **冗余** | 与其他用例高度相似（>85%）             | 合并后删除多余 |

## 相似用例检测

### 相似度计算

| 对比项     | 对应字段                                      | 权重 |
| ---------- | --------------------------------------------- | ---- |
| 用例名称   | `name`                                        | 30%  |
| 前置条件   | `pre_conditions`                              | 30%  |
| 步骤与预期 | `steps[].content` + `steps[].expected_result` | 40%  |

### 合并规则

1. 相似度 > 85% 的用例视为重复
2. 从相似用例组中选择**第一个用例（按uid排序）**作为保留目标
3. 合并后更新到保留用例，删除其他相似用例

### 合并内容策略

- **名称**：取最完整、最准确的名称
- **描述**：合并所有描述的关键信息，去重
- **前置条件**：取并集，去重
- **步骤**：合并所有不重复的测试步骤，按逻辑顺序排列
- **优先级**：取最高优先级（P0 > P1 > P2 > P3）
