## 项目概述

Plugin Finder 是一个智能插件发现和管理助手，旨在解决 CodeBuddy Code 用户在众多插件中难以找到和选择合适插件的问题。通过 AI 语义理解、智能推荐和多插件比较功能，帮助用户快速找到最适合他们需求的插件。

## 核心功能

### 1. 🔍 智能插件搜索
- **AI 语义匹配**：不仅是关键词搜索，而是理解用户意图
- **跨平台支持**：同时搜索 CodeBuddy 和 Claude 插件市场
- **多选安装**：一次选择多个插件，批量安装
- **相关度评分**：综合评估插件与需求的匹配度

### 2. 📦 便捷插件安装
- **批量安装**：一条命令安装多个插件
- **格式验证**：自动验证插件名称和市场名称
- **错误处理**：清晰的错误提示和解决建议
- **安装提醒**：自动提醒用户重启生效

### 3. 🔄 多插件比较
- **并行执行**：同时运行多个插件执行相同任务
- **多维分析**：质量、性能、输出、体验全方位比较
- **详细报告**：生成 Markdown 格式的对比报告
- **智能推荐**：基于分析结果推荐最佳插件

### 4. 🤖 智能自动推荐
- **需求识别**：自动识别用户可能需要插件的场景
- **主动推荐**：无需用户明确询问，智能推荐相关插件
- **可配置**：用户可控制推荐频率和阈值
- **非侵入式**：只在真正需要时才触发

### 5. 💡 插件许愿系统
- **需求收集**：找不到合适的插件时可以许愿
- **智能整理**：AI 自动整理和完善用户需求
- **邮件生成**：生成详细的许愿邮件草稿
- **直接反馈**：发送到 codebuddy@tencent.com
- **推动生态**：帮助 CodeBuddy 团队了解用户需求

## 关键设计决策

### 1. 语义匹配算法

**决策：** 使用 AI 语义理解 + 多维度评分

**理由：**
- 单纯关键词匹配效果差
- 用户表达方式多样
- 需要理解真实意图

**评分体系：**
- 描述匹配：50 分（主要）
- 关键词匹配：30 分
- 分类匹配：15 分
- 标签匹配：5 分

### 2. 双平台支持

**决策：** 同时支持 CodeBuddy 和 Claude

**理由：**
- CodeBuddy 基于 Claude Code
- 用户可能使用两个平台
- 提供更全面的插件覆盖

**实现方式：**
- 读取两个 known_marketplaces.json
- 统一的数据处理接口
- 兼容不同的数据格式

### 3. 自动推荐机制

**决策：** 使用 Hook + Agent 实现自动推荐

**理由：**
- 提升用户体验
- 主动发现用户需求
- 减少用户学习成本

**保护措施：**
- 可通过配置关闭
- 保守的触发策略
- 不影响正常对话

### 4. 多插件比较

**决策：** 并行执行 + 多维度分析

**理由：**
- 用户难以手动比较
- 节省用户时间
- 提供客观评价

**比较维度：**
- 质量（完整性、准确性、深度）
- 性能（执行时间）
- 输出（格式、文件）
- 体验（易用性）

## 创新点

### 1. AI 语义匹配
不同于传统的关键词搜索，使用 AI 理解用户真实需求。

### 2. 主动推荐
通过 Hook 监听用户输入，在合适时机主动推荐插件。

### 3. 智能比较
自动识别插件能力，选择最佳调用方式，并进行全面比较。

### 4. 用户可控
提供丰富的配置选项，让用户完全掌控插件行为。

## 挑战与解决方案

### 挑战 1：语义匹配准确性

**问题：** 如何确保推荐的插件真正相关？

**解决方案：**
- 使用 AI 模型进行语义理解
- 多维度综合评分
- 设置合理的推荐阈值
- 提供配置调整能力

### 挑战 2：避免过度推荐

**问题：** 自动推荐可能打扰用户？

**解决方案：**
- 保守的触发策略
- 明确的触发条件
- 用户可配置开关
- 只在真正需要时触发

### 挑战 3：多插件比较复杂性

**问题：** 如何公平地比较不同类型的插件？

**解决方案：**
- 识别插件能力（agent/command/skill）
- 使用相同的输入
- 多维度评估
- AI 综合分析

### 挑战 4：双平台数据格式差异

**问题：** CodeBuddy 和 Claude 数据格式不同？

**解决方案：**
- 统一的数据访问接口
- 兼容两种格式
- 优雅的降级处理
