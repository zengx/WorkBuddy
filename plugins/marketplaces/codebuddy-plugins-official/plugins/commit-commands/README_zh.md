# Commit Commands Plugin

通过简单的命令简化您的 git 工作流程，实现提交、推送和创建拉取请求。

## 概述

Commit Commands Plugin 自动化常见的 git 操作，减少上下文切换和手动命令执行。无需运行多个 git 命令，只需使用单个斜杠命令即可处理整个工作流程。

## 命令

### `/commit`

根据已暂存和未暂存的更改，自动生成提交消息来创建 git commit。

**功能说明：**
1. 分析当前 git 状态
2. 审查已暂存和未暂存的更改
3. 检查最近的提交消息以匹配您仓库的风格
4. 起草合适的提交消息
5. 暂存相关文件
6. 创建提交

**使用方法：**
```bash
/commit
```

**工作流程示例：**
```bash
# 对代码进行一些更改
# 然后只需运行：
/commit

# CodeBuddy 将：
# - 审查您的更改
# - 暂存文件
# - 创建带有合适消息的提交
# - 显示提交状态
```

**特性：**
- 自动起草符合您仓库风格的提交消息
- 遵循 conventional commit 规范
- 避免提交包含机密的文件（.env、credentials.json）
- 在提交消息中包含 CodeBuddy Code 署名

### `/commit-push-pr`

完整工作流程命令，一步完成提交、推送和创建拉取请求。

**功能说明：**
1. 创建新分支（如果当前在 main 分支）
2. 暂存并使用合适的消息提交更改
3. 将分支推送到 origin
4. 使用 `gh pr create` 创建拉取请求
5. 提供 PR URL

**使用方法：**
```bash
/commit-push-pr
```

**工作流程示例：**
```bash
# 进行更改
# 然后运行：
/commit-push-pr

# CodeBuddy 将：
# - 创建功能分支（如需要）
# - 提交您的更改
# - 推送到远程
# - 打开包含摘要和测试计划的 PR
# - 给您 PR URL 以供审查
```

**特性：**
- 分析分支中的所有提交（不仅仅是最新的）
- 创建包含以下内容的全面 PR 描述：
  - 更改摘要（1-3 个要点）
  - 测试计划清单
  - CodeBuddy Code 署名
- 自动处理分支创建
- 使用 GitHub CLI (`gh`) 创建 PR

**要求：**
- GitHub CLI (`gh`) 必须已安装并完成身份验证
- 仓库必须有一个名为 `origin` 的远程仓库

### `/clean_gone`

清理已从远程仓库删除的本地分支。

**功能说明：**
1. 列出所有本地分支以识别 [gone] 状态
2. 识别并删除与 [gone] 分支关联的 worktree
3. 删除所有标记为 [gone] 的分支
4. 提供有关已删除分支的反馈

**使用方法：**
```bash
/clean_gone
```

**工作流程示例：**
```bash
# 在 PR 合并并删除远程分支后
/clean_gone

# CodeBuddy 将：
# - 查找所有标记为 [gone] 的分支
# - 删除任何关联的 worktree
# - 删除过时的本地分支
# - 报告清理的内容
```

**特性：**
- 处理常规分支和 worktree 分支
- 在删除分支之前安全删除 worktree
- 显示有关已删除内容的清晰反馈
- 如果不需要清理则报告

**使用时机：**
- 在合并并删除远程分支后
- 当本地分支列表混乱时
- 定期仓库维护期间

## 安装

此插件包含在 CodeBuddy Code 仓库中。使用 CodeBuddy Code 时命令自动可用。

## 最佳实践

### 使用 `/commit`
- 在提交之前审查已暂存的更改
- 让 CodeBuddy 分析您的更改并匹配仓库的提交风格
- 信任自动生成的消息，但验证其准确性
- 用于开发期间的常规提交

### 使用 `/commit-push-pr`
- 准备创建 PR 时使用
- 确保所有更改都已完成并经过测试
- CodeBuddy 将分析完整的分支历史以生成 PR 描述
- 审查 PR 描述并根据需要进行编辑
- 当您希望最小化上下文切换时使用

### 使用 `/clean_gone`
- 定期运行以保持分支列表整洁
- 在合并多个 PR 后特别有用
- 运行安全 - 仅删除已在远程删除的分支
- 有助于保持本地仓库整洁

## 工作流程集成

### 快速提交工作流程：
```bash
# 编写代码
/commit
# 继续开发
```

### 功能分支工作流程：
```bash
# 跨多个提交开发功能
/commit  # 第一次提交
# 更多更改
/commit  # 第二次提交
# 准备创建 PR
/commit-push-pr
```

### 维护工作流程：
```bash
# 多个 PR 合并后
/clean_gone
# 清洁的工作空间准备好开发下一个功能
```

## 要求

- Git 必须已安装并配置
- 对于 `/commit-push-pr`：GitHub CLI (`gh`) 必须已安装并完成身份验证
- 仓库必须是带有远程仓库的 git 仓库

## 故障排除

### `/commit` 创建空提交

**问题**：没有更改可提交

**解决方案**：
- 确保您有未暂存或已暂存的更改
- 运行 `git status` 以验证更改存在

### `/commit-push-pr` 无法创建 PR

**问题**：`gh pr create` 命令失败

**解决方案**：
- 安装 GitHub CLI：`brew install gh` (macOS) 或参阅 [GitHub CLI 安装](https://cli.github.com/)
- 身份验证：`gh auth login`
- 确保仓库有 GitHub 远程仓库

### `/clean_gone` 找不到分支

**问题**：没有分支标记为 [gone]

**解决方案**：
- 运行 `git fetch --prune` 以更新远程跟踪
- 分支必须从远程删除才能显示为 [gone]

## 提示

- **与其他工具结合使用**：开发期间使用 `/commit`，准备就绪时使用 `/commit-push-pr`
- **让 CodeBuddy 起草消息**：提交消息分析从您仓库的风格中学习
- **定期清理**：每周运行 `/clean_gone` 以保持分支列表整洁
- **推送前审查**：推送前始终审查提交消息和更改
