# PPT内容生成技能

自动为PPT演示生成高质量的内容、大纲和演讲脚本。

## 激活场景

此技能在以下情况自动激活：

1. 用户需要创建PPT内容大纲
2. 用户要求生成演讲稿或备注
3. 用户需要扩展或丰富现有内容
4. 用户询问如何组织PPT逻辑
5. 用户需要案例、数据或引用支撑

## 核心能力

### 1. 内容结构设计

#### SCQA框架（麦肯锡金字塔原理）

**S (Situation) - 情境**：
- 描述当前背景和环境
- 建立与听众的共同认知
- 示例：`"在数字化转型的浪潮下..."`

**C (Complication) - 冲突**：
- 指出存在的问题或挑战
- 引发听众的关注和思考
- 示例：`"然而，80%的企业在转型中遇到瓶颈..."`

**Q (Question) - 问题**：
- 明确需要解决的核心问题
- 引出演讲的主题
- 示例：`"如何突破这些障碍，实现成功转型？"`

**A (Answer) - 答案**：
- 提供解决方案和核心观点
- 展开详细论述和案例
- 示例：`"我们的三步法可以帮助企业..."`

#### 三幕式结构

**第一幕：开场（10%）**
```
1. 引子：吸引注意（故事/数据/问题）
2. 背景：建立情境
3. 预告：概述主要内容
```

**第二幕：主体（70%）**
```
1. 核心观点1
   - 论据1
   - 论据2
   - 案例支撑
   
2. 核心观点2
   - 论据1
   - 论据2
   - 案例支撑
   
3. 核心观点3
   - 论据1
   - 论据2
   - 案例支撑
```

**第三幕：结尾（20%）**
```
1. 总结：回顾核心要点
2. 强化：重申关键信息
3. 行动：明确下一步
```

### 2. 标题撰写技巧

#### 动词驱动型标题
- `"提升效率的5个方法"` 而非 `"效率方法"`
- `"如何降低运营成本"` 而非 `"运营成本"`
- `"实现增长的关键策略"` 而非 `"增长策略"`

#### 数字化标题
- `"3个步骤优化用户体验"`
- `"5大趋势塑造未来市场"`
- `"7条原则打造高效团队"`

#### 问题式标题
- `"为什么我们需要改变？"`
- `"如何在竞争中脱颖而出？"`
- `"什么是成功的关键因素？"`

#### 对比式标题
- `"传统方法 vs 新型方案"`
- `"问题现状与理想状态"`
- `"Before & After：转型的力量"`

### 3. 内容展开技法

#### 金字塔原理
```
核心观点（结论先行）
├── 论据1
│   ├── 数据支撑
│   └── 案例说明
├── 论据2
│   ├── 数据支撑
│   └── 案例说明
└── 论据3
    ├── 数据支撑
    └── 案例说明
```

#### 5W2H分析法
- **What**：是什么（定义问题）
- **Why**：为什么（原因分析）
- **Who**：谁相关（利益相关方）
- **When**：何时（时间节点）
- **Where**：何地（场景范围）
- **How**：如何做（解决方案）
- **How much**：多少（成本收益）

#### STAR法则（讲故事）
- **Situation**：情境描述
- **Task**：任务挑战
- **Action**：采取行动
- **Result**：最终结果

### 4. 演讲备注生成

为每一页生成演讲备注，包含：

#### 开场白（第1页）
```
演讲时间：30秒
重点强调：吸引注意、建立联系
语气：热情、自信
技巧：
- 用一个引人入胜的故事或数据开场
- 与听众建立眼神交流
- 简要预告演讲内容
```

#### 内容页（中间页）
```
演讲时间：1-2分钟/页
重点强调：核心观点、关键数据
语气：专业、清晰
技巧：
- 先说结论，再展开论据
- 用具体案例说明抽象概念
- 适时提问，保持互动
- 关键数据重复强调
```

#### 总结页（最后页）
```
演讲时间：1分钟
重点强调：核心要点回顾、行动号召
语气：坚定、鼓舞
技巧：
- 简洁总结3-5个核心观点
- 明确下一步行动
- 以强有力的结语收尾
- 留出Q&A时间
```

### 5. 内容类型模板

#### 产品介绍PPT
```markdown
1. 封面：产品名称、Slogan
2. 问题：用户痛点和需求
3. 解决方案：产品如何解决问题
4. 核心功能：3-5个主要功能
5. 产品优势：与竞品对比
6. 应用场景：典型使用案例
7. 客户案例：成功故事
8. 技术架构：（技术受众）
9. 产品路线图：未来规划
10. 定价方案：套餐和价格
11. Q&A：常见问题
12. 行动号召：联系方式、试用入口
```

#### 项目汇报PPT
```markdown
1. 封面：项目名称、汇报人
2. 目录：汇报内容概览
3. 项目背景：目标和重要性
4. 项目进展：里程碑和完成度
5. 关键成果：量化数据展示
6. 遇到的挑战：问题和应对
7. 经验教训：总结和反思
8. 下一步计划：未来安排
9. 资源需求：人力、预算
10. 风险预警：潜在问题
11. 总结：关键要点回顾
12. Q&A：讨论和反馈
```

#### 培训课程PPT
```markdown
1. 封面：课程主题、讲师
2. 学习目标：学员将掌握什么
3. 课程大纲：内容模块预览
4. 知识点1：概念+案例
5. 实操演示：步骤展示
6. 练习1：巩固知识点1
7. 知识点2：概念+案例
8. 实操演示：步骤展示
9. 练习2：巩固知识点2
10. 综合案例：综合应用
11. 总结复习：关键要点
12. 拓展资源：延伸学习
```

#### 商业计划PPT
```markdown
1. 封面：公司名称、愿景
2. 问题机会：市场痛点
3. 解决方案：产品/服务
4. 市场分析：规模和趋势
5. 竞争格局：竞品分析
6. 商业模式：如何赚钱
7. 核心团队：创始人和高管
8. 发展里程碑：历史成就
9. 增长策略：市场拓展计划
10. 财务预测：收入和成本
11. 融资需求：金额和用途
12. 联系方式：投资对接
```

### 6. 内容增强技巧

#### 用数据说话
```
弱：我们的产品很受欢迎
强：我们的产品在过去6个月获得了10万用户，NPS评分达85分

弱：市场增长很快
强：市场年复合增长率达到35%，预计2025年规模将达到500亿美元

弱：我们节省了很多成本
强：通过流程优化，我们将运营成本降低了40%，每年节省200万元
```

#### 用案例增强说服力
```
模板：
[客户名称]遇到了[具体问题]，使用我们的[解决方案]后，
[时间周期]内实现了[具体成果]，[量化指标]提升了[百分比/数值]。

示例：
某大型零售企业面临库存周转率低的问题，部署我们的智能库存管理系统后，
3个月内库存周转率提升了50%，缺货率下降了30%，年度库存成本节省了800万元。
```

#### 用类比简化复杂概念
```
复杂：我们的系统采用分布式微服务架构...
类比：就像一个高效的快递网络，每个站点独立运作但协同配合

复杂：机器学习通过大量数据训练模型...
类比：就像教小孩识字，看得多了自然就会认

复杂：区块链是一种去中心化的分布式账本...
类比：就像全班同学都有一本账本，任何交易大家都同步记录
```

#### 用故事打动人心
```
故事结构：
1. 设置情境：介绍主人公和背景
2. 遇到挑战：描述问题或困境
3. 转折点：引入解决方案
4. 克服困难：展示行动过程
5. 结果和启示：说明影响和意义

示例：
"两年前，张经理的团队每周要花20小时手动整理销售报表，
经常加班到深夜。他们尝试过多种工具，但都不理想。
直到使用了我们的自动化报表系统，现在只需点击一个按钮，
5分钟就能生成精准的分析报告。张经理说：
'这个工具让我们重新找回了工作的意义，有更多时间服务客户。'"
```

### 8. 常见内容问题和解决方案

#### 问题1：内容空洞
**表现**：大量空话套话，缺少实质内容
**解决**：
- 每个观点都有数据或案例支撑
- 用具体事例替代抽象描述
- 提供可操作的建议
- 量化所有可量化的内容

#### 问题2：逻辑混乱
**表现**：内容跳跃，前后不连贯
**解决**：
- 使用SCQA或三幕式结构
- 明确每页的核心观点
- 添加过渡页承上启下
- 确保整体叙事流畅

#### 问题3：信息过载
**表现**：想说的太多，听众记不住
**解决**：
- 聚焦3-5个核心观点
- 删减次要信息
- 一页一个重点
- 使用"Rule of Three"原则

#### 问题4：缺少互动
**表现**：单向输出，听众注意力下降
**解决**：
- 在关键位置设置提问
- 邀请听众分享经验
- 设计小测验或投票
- 预留讨论时间

#### 问题5：结尾无力
**表现**：草草收场，缺少行动号召
**解决**：
- 明确总结核心要点
- 提出清晰的下一步行动
- 用强有力的结语收尾
- 留下深刻印象

## 内容生成流程

### 第一步：需求分析
```
询问用户：
1. PPT主题和目标是什么？
2. 目标听众是谁？（背景、职位、需求）
3. 演讲场合和时长？
4. 希望传达的核心信息？
5. 有没有必须包含的内容？
```

### 第二步：结构设计
```
根据需求设计结构：
- 确定总页数（根据演讲时长）
- 划分章节（3-5个核心主题）
- 分配页数（每个主题2-5页）
- 设计过渡（章节间的衔接）
```

### 第三步：内容撰写
```
逐页生成内容：
- 撰写页面标题（简洁有力）
- 提炼核心观点（一句话）
- 展开支撑要点（3-5个）
- 添加数据案例（具体量化）
- 编写演讲备注（时间+要点）
```

### 第四步：优化迭代
```
检查和改进：
- 整体逻辑是否清晰？
- 信息量是否适中？
- 重点是否突出？
- 案例是否充分？
- 语言是否精炼？
```

## 快速内容生成技巧

### 10分钟生成大纲
1. 确定核心信息（30秒）
2. 列出3-5个观点（2分钟）
3. 每个观点展开3个要点（5分钟）
4. 添加开场和结尾（2分钟）
5. 检查逻辑流畅性（30秒）

### 内容扩展套路
- **What + Why + How**：是什么 → 为什么重要 → 如何实现
- **Problem + Solution + Benefit**：问题 → 解决方案 → 好处
- **Past + Present + Future**：过去 → 现在 → 未来
- **Theory + Example + Practice**：理论 → 案例 → 实践

### 应急内容方案
时间紧迫时：
1. 明确3个核心观点
2. 每个观点找1个数据或案例
3. 开场用提问或数据吸引注意
4. 结尾总结观点并号召行动
5. 确保逻辑连贯即可
