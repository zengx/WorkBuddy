# PPT创建命令

根据指定主题创建完整的PPT演示文稿大纲和内容。

## 使用方式

```bash
/create-ppt "人工智能发展趋势" --slides 15 --template tech --format markdown
```

## 执行步骤

你将帮助用户创建一个专业的PPT演示文稿：

### 1. 分析主题和需求

- 理解用户提供的主题：{{topic}}
- 确定目标受众和演讲场景
- 明确PPT的核心目标和关键信息

### 2. 生成PPT结构

根据幻灯片数量（{{slides}}），创建清晰的PPT结构：

**标准结构**：
1. **封面页** - 标题、副标题、演讲者信息
2. **目录页** - 整体框架预览
3. **引言** - 背景介绍、问题提出
4. **主体内容** - 3-5个核心章节，每章节2-4页
5. **总结页** - 核心要点回顾
6. **致谢/Q&A页** - 结束语

### 3. 生成每页内容

为每一页生成：

**标题**：简洁有力（5-8字）
**核心观点**：1-2句话的关键信息
**要点列表**：3-5个支撑要点（每个10-20字）
**视觉建议**：
  - 推荐图表类型（柱状图/饼图/流程图等）
  - 配色方案建议
  - 图标或图片建议

**演讲备注**：每页30-50字的演讲提示

### 4. 应用模板风格

根据选择的模板类型（{{template}}）调整风格：

- **business**（商务）：专业、简洁、数据驱动
  - 配色：深蓝、灰色、白色
  - 字体：正式、易读
  - 重点：数据图表、案例分析

- **tech**（技术）：现代、创新、视觉化
  - 配色：科技蓝、黑色、荧光色
  - 字体：现代、科技感
  - 重点：流程图、架构图、代码示例

- **education**（教育）：友好、清晰、循序渐进
  - 配色：温暖色系、高对比
  - 字体：清晰、易读
  - 重点：步骤图、知识点、练习

- **minimal**（极简）：简约、留白、聚焦
  - 配色：单色或双色
  - 字体：大字号、粗体
  - 重点：一页一个核心概念

### 5. 输出格式化内容

根据指定格式（{{format}}）输出：

#### Markdown格式（默认）：
```markdown
# PPT标题：{{topic}}

## 第1页：封面
**标题**：{{topic}}
**副标题**：[生成相关副标题]
**演讲者**：[待填写]

## 第2页：目录
1. [章节1名称]
2. [章节2名称]
3. [章节3名称]
...

## 第3页：[页面标题]
**核心观点**：[一句话核心信息]

**要点**：
- 要点1
- 要点2
- 要点3

**视觉建议**：
- 图表类型：[图表类型]
- 配色：[配色方案]
- 图标：[图标建议]

**演讲备注**：[演讲提示]

---
[继续其他页面...]
```

#### JSON格式：
```json
{
  "title": "{{topic}}",
  "template": "{{template}}",
  "totalSlides": {{slides}},
  "slides": [
    {
      "number": 1,
      "type": "cover",
      "title": "标题",
      "subtitle": "副标题",
      "notes": "演讲备注"
    },
    {
      "number": 2,
      "type": "content",
      "title": "页面标题",
      "keyMessage": "核心观点",
      "bullets": ["要点1", "要点2", "要点3"],
      "visual": {
        "chartType": "bar",
        "colors": ["#1a73e8", "#34a853"],
        "icons": ["chart-icon"]
      },
      "notes": "演讲备注"
    }
  ]
}
```

#### HTML格式：
生成简单的HTML演示文稿，可在浏览器中查看。

### 6. 提供优化建议

生成内容后，提供以下建议：

**内容优化**：
- 哪些页面可以合并或拆分
- 逻辑流是否清晰
- 是否缺少关键信息

**视觉优化**：
- 配色是否协调
- 信息密度是否合适
- 视觉层次是否清晰

**演讲技巧**：
- 时间分配建议
- 重点强调部分
- 互动环节建议

## 注意事项

1. **内容质量**：
   - 确保每页信息量适中（3-5个要点）
   - 避免大段文字，使用简洁表述
   - 数据和案例要有说服力

2. **视觉平衡**：
   - 每页不超过3种颜色
   - 字体大小层次分明
   - 留白充足，避免拥挤

3. **逻辑连贯**：
   - 页面之间有清晰的过渡
   - 整体叙事流畅
   - 首尾呼应

4. **实用性**：
   - 根据演讲时长调整内容深度
   - 预留互动和Q&A时间
   - 提供可操作的建议和结论

## 示例

用户输入：
```
/create-ppt "企业数字化转型" --slides 12 --template business
```

你应该生成：
- 12页完整的PPT大纲
- 每页包含标题、要点、视觉建议
- 商务风格的配色和布局建议
- 演讲备注和优化建议

最终输出一份可直接用于制作PPT的详细文档。
